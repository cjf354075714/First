Dubbo 的架构：
分为5个节点，这五个节点就代表了一个项目中的，对项目功能角色的描述
1，registry，注册中心，典型的服务注册与发现的中心
2，consumer，服务的消费者，就是调用别人服务的服务，在初始化的时候，会订阅注册中心，注册中心也会异步通知服务消费者，
服务消费者的消费是通过同步的方式来实现的
3，provider，服务的提供者，需要向注册中心注册自己的存在，并运行在 container 中，代表着一个服务，向消费者提供同步的消费服务
4，monitor，管理者，管理着服务调用时间、次数等信息
5，container，服务提供者的运行容器

Dubbo 的特点：
连通性：
注册中心与消费者和提供者只在启动的时候，出现一个信息交互，注册中心不转发请求
监控中心监控所有服务调用与请求
？
？
注册中心、消费者、提供者三者之间均为长连接，什么是长连接，这里就是 tcp 连接
消费者可以缓存提供者的地址
注册中心和监控中心可选
健壮性：
使用缓存，使用集群，自动切换等
伸缩性：
注册中心和服务提供者的新增都可以动态增加
